# git与github配置ssh教程

为了证明你是你，你的电脑上要有个绝密文件，还有个和它配套的不绝密的文件

把那个不绝密的文件拿一份给github，然后就可以保证是你这台电脑在跟github通信了

## 生成key

如何获得这两个文件：随便哪开个命令行，输入ssh-keygen运行，开始生成俩文件

会要你设个密码，反正我是设了，这样别人就算偷到了绝密文件也得麻烦半天去猜密码，不设置的话你把文件保管好就行（不能被人扫盘）

结束了后就会在你的用户目录（生成的时候能改但不推荐，大多数应用默认从这读取）内多出.ssh这个文件夹

里面id_rsa是绝密文件，id_rsa.pub是不绝密文件

## 配置github

打开github个人设置-SSH & GPG keys，点击右上角New SSH key，会出来个大框

把那个xxx.pub文件用记事本打开，复制到上面那个大框里，标题随便取一个，比如取个生成key用的邮箱之类，然后github就认识你（的电脑）了！

## git使用

然后这个身份证明怎么用呢，git pull默认用https的啊

为啥是https？你可能没注意，git clone的时候用的那个链接就是https://打头

用类似ssh://这个意思的东西就可以了！用这个链接的话系统就会使用你的绝密文件（不是直接发出去）加密数据再发送，有pub文件的人（这里是github）才能解密并确认是你

这个链接在仓库那个绿色的Code按钮里，点开选择SSH就是了

如果你已经clone过https的了也没关系，其实那个https://只是存在本地的链接，改一下就行：

`git remote --set-url origin 新链接`

然后你就可以用ssh访问github上的仓库啦！
